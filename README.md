## 프로젝트 소개

“손님 앞에서는 늘 웃지만,
그 뒤에는 복잡한 스케줄과 교대표가 기다리고 있었죠.”

하루에도 수십 번 바뀌는 근무표,
휴무 신청할 때마다 반복되는 카톡 보고,
교대 요청 하나 하려면 몇 명에게 말을 걸어야 했던 직원들.

그래서, 메가박스 직원의 목소리에서 시작된 진짜 변화가 만들어졌습니다.
우리가 만드는 것은 단순한 관리 시스템이 아닙니다.
직원들의 하루가 조금 더 편해지는 경험, 바로 그것입니다. 🌟

“현장의 불편함을 가장 잘 아는 사람이 직접 기획하고,
개발자들이 함께 해결해가는 프로젝트.”

이건 “관리”를 위한 관리가 아니라,
“사람을 위한 관리 시스템”이에요. 🎥

---

## 주요 기능

- **직원 로그인 & 권한 관리**  
  안전하고 간편한 로그인으로 개인화된 근무 환경 제공
- **관리자 페이지**  
  직원 정보, 스케줄, 근무 기록 등 효율적인 통합 관리
- **스케줄 & 근무교대 페이지**  
  교대 요청부터 승인까지 한 번에 처리
- **휴무 신청 페이지**
  휴무 신청과 승인 진행
- **급여 관리 페이지**  
  근무 내역 기반 자동 계산 및 기록 확인
- **커뮤니티 페이지**  
  직원 간의 소통과 공지 공유의 공간

## 📑 프로젝트 규칙

### Branch Strategy

> - main / dev 브랜치 기본 생성
> - 각 application 단위마다 feature 브랜치 생성(ex. feature/user/#issuenumber)
> - main과 dev로 직접 push 제한
> - PR 전 최소 1인 이상 승인 필수

### Git Convention

> 1. 적절한 커밋 접두사 작성
> 2. 커밋 메시지 내용 작성

> | 접두사     | 설명                           |
> | ---------- | ------------------------------ |
> | Feat :     | 새로운 기능 구현               |
> | Add :      | 에셋 파일 추가                 |
> | Fix :      | 버그 수정                      |
> | Docs :     | 문서 추가 및 수정              |
> | Style :    | 스타일링 작업                  |
> | Refactor : | 코드 리팩토링 (동작 변경 없음) |
> | Test :     | 테스트                         |
> | Deploy :   | 배포                           |
> | Conf :     | 빌드, 환경 설정                |
> | Chore :    | 기타 작업                      |

### Pull Request

> ### Title
>
> - 제목은 '#96 Feat: 홈 페이지 구현'과 같이 작성합니다.
> - 제목 맨 앞에 '#96' 과 같은 번호는 이슈 번호를 의미합니다.

> ### PR Type
>
> - [ ] FEAT: 새로운 기능 구현
> - [ ] ADD : 에셋 파일 추가
> - [ ] FIX: 버그 수정
> - [ ] DOCS: 문서 추가 및 수정
> - [ ] STYLE: 포맷팅 변경
> - [ ] REFACTOR: 코드 리팩토링
> - [ ] TEST: 테스트 관련
> - [ ] DEPLOY: 배포 관련
> - [ ] CONF: 빌드, 환경 설정
> - [ ] CHORE: 기타 작업

### Communication Rules

> - Discord 활용
> - notion 활용
